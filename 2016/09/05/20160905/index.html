Redmine是一个基于项目和事务跟踪系统，通过它我们可以完成任务指派、跟踪、监督>等工作。

Redmine在Ubuntu 16.04.01上的安装

安装redmine所需要的软件

redmine是基于RoR开发的，因此首先需要安装ruby和rails，同时为了通过MySQL来记录我的们数据，因此需要安装mysql。如果是从redmine的网站下载redmine软件包的话就不需要安装git，不过建议安装，因为到后面会用到git来做版本管理。安装imagemagick和libmagickwand-dev则是为了解决后期安装rmagick的时候所需要依赖包问题而提前安装的。libmysqlclient-dev和libmysqlclient20是mysql2的依赖安装。

apt-get install mysql-server git ruby rubygems rails bundler imagemagick libmagickwand-dev libmysqlclient-dev libmysqlclient20

安装过程中会提示设置MySQL的管理员密码，一定要记住了。

使用Git下载Redmine

使用下面的命令，时间不长，一会就下载完了。如果需要下载Redmine的包的话可以到官方网站：https://www.redmine.org/projects/redmine/wiki/Download上下载。最新的版本是3.3.0

cd /srv/
git clone https://github.com/redmine/redmine

配置Redmine数据库

使用下面的命令来建立数据库和用户，其中xxxxxx是新建立的redmine用户的密码。

$mysql -u root -p

mysql> CREATE DATABASE redmine CHARACTER SET utf8;
mysql> CREATE USER 'redmine'@'localhost' IDENTIFIED BY 'xxxxxx';
mysql> GRANT ALL PRIVILEGES ON redmine.* TO 'redmine'@'localhost';
mysql> exit

建立完成数据库后可以来配置Redmine数据库文件了。

cd /srv/redmine
cp  config/database.yml.example config/database.yml
vi config/database.yml

修改config/database.yml文件，将你的用户名和密码写入到相应的字段中：

production:
  adapter: mysql
  database: redmine
  host: localhost
  username: redmine
  password: "xxxxxx"
  encoding: utf8

调整一下gem的源

gem sources --add https://ruby.taobao.org/ --remove https://rubygems.org/
bundle config mirror.https://rubygems.org https://ruby.taobao.org

gem默认使用国外http://rubygems.org/的源，为了更快的下载软件，我们将它调整为淘宝的源：https://ruby.taobao.org

Redmine的bundle依赖文件安装

cd /srv/redmine
bundle install

bundle会根据Redmine的需要安装依赖文件，整个时间比较长，慢慢等吧。

Redmine的配置

rake generate_secret_token
RAILS_ENV=production rake db:migrate

RAILS_ENV=production rake redmine:load_default_data

上面最后一个命令是选择语言，手动输入一个zh吧。

# rake generate_secret_token

如果提示以上错误可以直接编辑/var/lib/gems/2.3.0/gems/htmlentities-4.3.1/lib/htmlentities/mappings/expanded.rb文件，将第466行注释掉就行了。

建立Redmine的文件目录及设置文件权限

#mkdir -p tmp tmp/pdf public/plugin_assets
#chmod -R 755 files log tmp public/plugin_assets

建立包括临时文件夹什么目录，并给上相应的权限

Redmine测试

至此Redmine就安装完成了，现在就可以使用下面的命令进行测试了。

#ruby bin/rails server webrick -e production

=> Booting WEBrick
=> Rails 4.2.7.1 application starting in production on http://localhost:3000
=> Run `rails server -h` for more startup options
=> Ctrl-C to shutdown server
[2016-09-05 15:03:50] INFO  WEBrick 1.3.1
[2016-09-05 15:03:50] INFO  ruby 2.3.1 (2016-04-26) [x86_64-linux-gnu]
[2016-09-05 15:03:50] INFO  WEBrick::HTTPServer#start: pid=22987 port=3000

运行上面的服务之后，我们就可以在浏览器中输入http://localhost:3000 来测试。如果安装成功就会出现Redmine的网站界面。默认的管理员帐号和密码都是”admin“。

如果要长期使用可以加上-d参数，不过我更习惯使用screen来运行它。

安装Apache2

Redmine是使用rails框架开发的，默认只能使用http://localthost:3000来访问，当然你也可以设置一下让他直接访问，不过为了更好的性能，我们来通过apache2来进行反向代理,并通过缓存模块来提速。

#apt-get install apache2

为Apache2启用反向代理模块

ln -sf /etc/apache2/mods-available/proxy.load /etc/apache2/mods-enabled//proxy.load
 ln -sf /etc/apache2/mods-available/proxy_http.load /etc/apache2/mods-enabled/proxy_http.load

修改虚拟主机的配置文件，增加反向代理参数：

vi sites-enabled/000-default.conf

增加以下参数，端口号后面一定要加”/“，否则会无法正常代理的。

ProxyVia Off
ProxyRequests On
ProxyPreserveHost On
ProxyPass / http://localhost:3000/
ProxyPassReverse / http://localhost:3000/

现在重新启动Apache服务器，即可正常使用浏览器来访问Redmine了。

如果追求更好的性能可以开启Apache的cache缓存模块来提高速度。
如果用户连接多，可以调整一下Apache的mpm模块。
